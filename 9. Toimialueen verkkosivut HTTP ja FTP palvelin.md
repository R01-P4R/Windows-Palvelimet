##  IIS- ja FTP-palvelun asennus

<br>

Tehtävässä oli tarkoitus asentaa IIS- ja FTP-palvelu toimialueen tiedostopalvelimelle (WindowsServer_Fileserver). IIS on Microsoftin kehittämä verkkopalvelinohjelmisto, joka pystyy jakamaan mm. verkkosivuja. FTP-palvelu on yksi IIS:n roolipalveluista (Role Service), mikä mahdollistaa tiedostojen siirtämisen. Aloitin asentamalla FTP-palvelun tiedostopalvelimen Server Managerin kautta valikosta "Manage" ja "Add Roles and Features". Avautuvan asennusohjelman avulla määritin ensin tiedostopalvelimeni asennuksen kohteeksi ja valitisin halutun roolin (Web Server (IIS)). Roolipalveluiden oletusvalintojen lisäksi valitsin myös "FTP Server", "FTP Service" ja "HTTP Redirection".

## IIS-testaus

<br>

Alotin testaamalla, että IIS-palvelu asentui avaamalla tiedostopalvelimen selaimen ja siirtymällä osoitteeseen:

    http://127.0.0.1/

Loopback-osoitteessa näkyi IIS:n testisivu. Seuraavaksi avasin tiedostopalvelimen Server Managerin kautta IIS:n oman hallintatyökalun valikosta "Tools" ja "Internet Information Services (IIS) Manager". Aloitin ensin pysäyttämällä oletusverkkosivun kohdasta "Default Web Site" ja "Stop". 
<br>

 <img width="1151" height="651" alt="image" src="https://github.com/user-attachments/assets/8a5fe093-7c16-4b24-b09b-4cb5a43d4ef6" />

<br>

Tämän jälkeen kävin kopioimassa oletusverkkosivun sisältörakenteen polusta "C:\inetpub" luodakseni uusia verkkosivuja helposti. Loin ensin uuden kopion nimeltä "inettesti" polkuun "C:\inettesti". Seuraavaksi siirryin IIS Manageriin ja valitsin Sites-valikosta "Add Website...".

<img width="1044" height="775" alt="image" src="https://github.com/user-attachments/assets/0f50bc46-285f-439b-b1a8-99cea7d844ef" />

<br>

Käytin seuraavia arvoja:


      Site name:          Testi
      Physical path:      C:\inettesti\wwwwroot
      Binding - Type:      http

Lisäksi valitsin myös "Start Website immediately", jotta sivua ei tarvitse erikseen käynnistää IIS Managerin kautta. Hyväksyttyä valintani varoitettiin portin 80 olevan jo käytössä toisella sivulla (Default Web Site) ja haluanko pitää sen myös tällä sivulla (vain jompaakumpaa sivua voi käyttää kerrallaan samassa portissa). Valitsin kyllä ja siirryin muokaamaan testisivun ulkonäköä polusta: "C:\inettesti\wwwroot\iistart.htm". Muokkasin uuden sivun HTLM-sisältö notepadia hyödyntäen ja lisäsin seuraavan uuden elementin:

      <div>
         <h1 style="font-family: Arial; position: absolute; top: 75px; left: 500px">
            TEST 1
         </h1>
      </div>


<br>

Tallenettua muutokset sivun sisältöön siirryin tiedostopalvelimen selaimella osoitteeseen: 

      http://localhost/

Eri tapa viitata samaan loopback-osoitteeseen (http://127.0.0.1/).

## Tietueiden ja käänteisvyöhykkeen lisäys
   
<br>

Aloitin luomaan kahta alias-nimeä (CNAME) suoran haun vyöhykkeelleni (Forward Lookup Zone), jotka viittaavat tiedostopalvelimeni nimeen. Loin alias-nimet ohjauspalvelimeni DNS Managerissa ensin valitsemalla vyöhykkeeni "partanen.lan" ja sitten "New Alias (CNAME)...". Loin kasi uutta CNAME-tietuetta:

      web.partanen.lan
      intra.partanen.lan

Määrittelemällä ensin Alias-nimen (intra tai web) ja sen jälkeen valitsemalla haun avulla (Browse...) tiedostopalvelimeni.


<img width="1064" height="513" alt="image" src="https://github.com/user-attachments/assets/1df1ef16-f283-442c-b484-e3f74c0bca48" />

> Luodut uudet alias-nimet (CNAME). tiedostopalvelimelle.

<br>

Käänteisen haun vyöhykkeen (Reverse Lookup Zone) lisäsin DNS Managerin kautta kohdasta: "Reverse Lookup Zones" ja "New zone...". Käänteisen haun vyöhykkeiden tehtävä on selvittää IP-osoitetta vastaava domain-nimi, esim. kun suorittaa nslookup-komennon IP-osoitteelle. Käänteisvyöhykkeiden luomiseen oli apuna asennusohjelma, jonka avulla pääsi määrittämään haluamansa asetukset. Valitsin: 

      Zone type:                               Primary zone 
      Store the zone in Active Directory:      Yes
      Zone data replication:                   To all DNS servers running on domain controllers in this domain
      Reverse Lookup Zone name:                IPv4 Reverse Lookup Zone 
      Network ID:                              Hyper-V verkon verkkoavaruuden arvo: 10.208.0
      Dynamic Update:                          Allow only secure dynamic updates

<br>

PTR-tietueita käytetään käänteisissä nimenselvityspyynnöissä kertomaan kohteen nimi. Uudelle käänteisen haun vyöhykkeelle loin PTR-tietueet ohjauspalvelimelle ja tiedostopalvelimelle. valitsin ensin luodun käänteisen haun vyöhykkeen "0.208.10.in-addr.arpa" ja sitten "New Pointer (PTR)...". Host IP Address-kohtaan oli syötettävä halutun tietokoneen IPv4-osoitteen host-osa, minun tapauksessa 10 ja 12.

## A- ja PTR-tietueet


Lisäsin uudet A-tietueet uudelle suoran haun vyöhykkeelle: "PARROI.COM" valitsemalla sen DNS Managerista ja "New Host (A or AAAA)...". Loin seuraavat A-tietueet:



      Name      IP address
      HOST-1    10.120.41.10 
      HOST-2    10.120.41.20 
      HOST-3    10.120.41.30 



<br>

Valitsin myös "Create associated pointer (PTR) record", mikä luo käänteisvyöhykkeen PTR-tietueet automaattisesti. Vastauksena tehtävän lisäkysymykseen: "Tiedätkö, miksi mikään koneista ei ole osoitteessa 10.120.41.1?", yleensä standardin mukaisesti verkon ensimmäinen käytettävissä oleva osoite (tässä tapauksessa 10.120.41.1) on varattu reitittimelle (Default Gateway-oletusyhdyskäytävä).
