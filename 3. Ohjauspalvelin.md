## Ohjauspalvelimen alkumääritykset

Aloitin yhdistämällä luotuun ohjauspalvelimeen "WindowsServer_DC" Hyper-V managerin kautta. Koneen käynnistettyä valitsin maa- ja näppäimistö asetuksiksi "Finland" ja "Finnish". Kielen muutin "English (United States), sillä IT-maailmassa se on ns. de facto-kieli. Seuraavaksi hyväksyin lisenssiehdot, määritin ylläpitäjän salasanan kurssiohjeiden mukaisesti ja kirjauduin virtuaalikoneen työpöydälle. 


## Ohjauspalvelimen staattinen IP-osoite

Avasin virtuaalikoneen WindowaServer_DC työpöydälle Server Manager-työkalun ja valikosta "Local Server" valitsin Ethernetin vierestä "IPv4 address assigned by DHCP, IPv6 enabled". Sitten valitsin Ethernet-verkkokortista "Properties" ja valitsin "Internet Protocol Version 4 (TCP/IPv4)" Tästä valikosta valitsin vielä "Properties" ja määritin ohjeen mukaisesti IPv4-tiedot (IP-addess, Subnet mask, Default Gateway ja DNS-asetukset) seuraavanlaisesti:

    IP address: 10.208.0.10
    Subnet mask: 255.255.255.0
    Default Gateway: 10.208.0.1
    Preferred DNS sever: 168.63.129.16

Nämä asetukset määritettyä sain Windowsilta ilmoituksen uuden verkon luomisesta ja verkkotilatyypin määrityksestä. Valitsin verkkotilaksi yksityinen.
Lopuksi testasin verkkoyhteyden toimivuutta cmd:en kautta komennolla "nslookup haaga-helia.fi". Sain toivetun tuloksen.

## Ohjauspalvelimen nimi ja lukitusruudun poistaminen käytöstä

Aloitin valitsemlla virtuaalikoneen WindowsServer_DC Server Managein ja valikosta Local Server valitsin luodon tietokoneen nimen. Kohdasta "Computer Name" valitsin "Change..." ja määritin ohjeiden mukaisesti uuden nimen "Controller-Partanen" ohjauspalvelimelle. Nimen muutoksen jälkeen oli virtuaalikoneen käynnistettävä uudestaan.

## Ohjauspalvelimen käyttöjärjestelmän aktivointi

Aloitin avaamalla virtuaalikoneen WindowsServer_DC:n komentorivin (cmd) ylläpitäjän oikeuksin ja suorittamalla komennon "slmgr.vbs", joka antaa infoa Windows käyttöjärjestelmän aktivoitamiseen käytettävän työkalun slmgr.vbs käytöstä. Läpikäytyäni tiedot suoritin komennon "slmgr.vbs /dlv" saadakseni tietää nykyaktivoinnin tilan. Tämän jälkeen suoritin komennon "slmgr /skms kms.core.windows.net" kertoakseni slmgr.vbs-työkalulle missä KMS-aktivointipalvelin sijaitseen. Tämän jälkeen pääsin suorittamaan aktivointipyyntöä komennolla "slmgr /ato", jonka jälkeen sain ilmoituksen, että aktivointi onnistui.

## Ohjauspalvelimen palvelinohjelmisto

Aloitin käynnistämällä virtuaalikoneen WindowsServer_DC:n ja Server Managerin kautta valitsin oikeasta yläkulmasta "Manage" ja "Add Roles and Features". Avautuvan infon luettuani valitsen "Role-based or feature-based installation" ja valitsin listalta ohjauspalvelimen Controller-Partanen. Valitsin tarjottavista rooleista "Active Directory Domain Services" valitsin myös seruaavasta ikkunasta "Include management tools" ja siirryin kohtaan "Add Features". Fetures kohdasta siirryin oletusasetuksilla eteenpäin. Sain tietoa asennuksessa valitusta roolista ja yhteenvedon valituista sennuksista. Lopuksi varmistin asennuksen ja virtuaalikone käynnistyi uudestaan.

<br>
<br>

Asennuksen onnistettua avasin Server Managerin ja uuden valikon "AD DS" alta valitsin "Controller-Partanen". Valitsin kohdasta "More..." -> "Promote this server to a domain controller". Aukeavassa valikossa pääsin luomaan aktiivihakemistolle uutta ylintä tasoa eli "forest", valitisn "Add a new forest" ja asetin sen nimeksi "partanen.lan". Valitsin seuraavaksi Windows Server 2016-tason, sekä valitsin "Domain Name System (DNS) Server", eli ohjauspalvelimella Controller-Partanen on nimipalvelimen (DNS) rooli. DSRM-salasankasi asetin ohjeen mukaan. DSRM on toimnto jolla kone voidaan käynnistää, vaikka aktiivihakemistoon tulisi häiriö. NetBios-toimialeen mimeksi valitsin päätoimialueen nimen ensimmäisen osan isolla kirjoitettuna, eli "PARTANEN". Seuraavan ikkunan polkuvaihtoehdot pidin oletuksena. Asennuksen lopuksi tietokone käynnistyi uudestaan.
