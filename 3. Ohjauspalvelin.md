# 3. Ohajauspalvelin

## Tehtävänanto

1. Luodaan ohjauspalvelin toimialueelle
2. Asennetaan Windows-käyttöjärjestelmä luotuun virtuaalikoneeseen
3. Luodun virtuaalikoneen mallinnus
4. Mallikoneen käyttäminen uusien virtuaalikoneiden pohjana

##  Ohjauspalvelimen rungon määritys

Aloitin kirjautumalla Azure Labsiin ja avaamalla virtuaalikoneen. Seuraavaksi ryhdyin luomaan uutta virtuaalikonetta hyödyntäen aikaisemmassa harjoituksessa luotua virtuaalikovalevyä "WindowsServer_DC.vhdx". Avasin Hyper-V managerin ja valitsin "New -> Virtual Machine. 

<img width="957" height="936" alt="image" src="https://github.com/user-attachments/assets/35920a2e-893c-42c9-bb3d-033af9b796b5" />

Annoin uudelle virtuaalikoneelle nimeksi "WindowsServer_DC" ja valitsin ensimmäisen generaation. Muistin määräksi asetin 3072 MB ja muistin dynaamisen allokoinnin laitoin päälle. Verkkoyhteden adapteriksi valitisn aikaisemmin luodon HyperVVSwitch ja virtuaalikovalevyksi valitsin harjoituksessa 2. Virtuaalikoneiden malli ja palvelinkoneiden alustus luodon WindowsServer_DC.vhdx templaten. Lopuksi vielä muutin Hyper-V managerin kautta käytettyjen virtuaaliprosessorien ytimien määräksi 4.

## Ohjauspalvelimen määritys

Aloitin yhdistämällä luotuun ohjauspalvelimeen "WindowsServer_DC" Hyper-V managerin kautta. Koneen käynnistettyä valitsin maa- ja näppäimistö asetuksiksi "Finland" ja "Finnish". Kielen muutin "English (United States), sillä IT-maailmassa se on ns. de facto-kieli. Seuraavaksi hyväksyin lisenssiehdot, määritin ylläpitäjän salasanan kurssiohjeiden mukaisesti ja kirjauduin virtuaalikoneen työpöydälle. Tästä valikosta valitsin vielä "Properties" ja määritin ohjeen mukaisesti IPv4-tiedot (IP-addess, Subnet mask, Default Gateway ja DNS-asetukset) seuraavanlaisesti:

    IP address: 10.208.0.10
    Subnet mask: 255.255.255.0
    Default Gateway: 10.208.0.1
    Preferred DNS sever: 168.63.129.16

Nämä asetukset määritettyä sain Windowsilta ilmoituksen uuden verkon luomisesta ja verkkotilatyypin määrityksestä. Valitsin verkkotilaksi yksityinen.
Lopuksi testasin verkkoyhteyden toimivuutta cmd:en kautta komennolla "nslookup haaga-helia.fi". Sain toivetun tuloksen.

## Ohjauspalvelimen staattinen IP-osoite

Avasin virtuaalikoneen työpöydälle Server Manager-työkalun ja valikosta "Local Server" valitsin Ethernetin vierestä "IPv4 address assigned by DHCP, IPv6 enabled". Sitten valitsin Ethernet-verkkokortista "Properties" ja valitsin "Internet Protocol Version 4 (TCP/IPv4)" 

## Ohjauspalvelimen nimi ja lukitusruudun poistaminen käytöstä

Aloitin valitsemlla virtuaalikoneen WindowsServer_DC Server Managein ja valikosta Local Server valitsin luodon tietokoneen nimen. Kohdasta "Computer Name" valitsin "Change..." ja määritin ohjeiden mukaisesti uuden nimen ohjauspalvelimelle. Nimen muutoksen jälkeen oli virtuaalikoneen käynnistettävä uudestaan.

<br>
<br>

Seuraavaksi siirryin poistamaan lukitusruutu-ominaisuutta pois käytöstä, sillä se voi aiheuttaa hankaluuksia virtuaalikoneille, millä ei ole käytössä fyysistä näppäimistöä.
Lukitusruutu-ominaisuuden sain pois päältä käyttämällä ryhmäkäytäntöohjelmaa gpedit.mcs. Valitsin Local Computer Policy -> Computer Configuration -> Administrative Templates -> Control Pane -> Personalization -> Do not display the lock screen ja "Enabled".

## Ohjauspalvelimen käyttöjärjestelmän aktivointi

Aloitin avaamalla virtuaalikoneen WindowsServer_DC:n komentorivin (cmd) ylläpitäjän oikeuksin ja suorittamalla komennon "slmgr.vbs", joka antaa infoa Windows käyttöjärjestelmän aktivoitamiseen käytettävän työkalun slmgr.vbs käytöstä. Läpikäytyäni tiedot suoritin komennon "slmgr.vbs /dlv" saadakseni tietää nykyaktivoinnin tilan. Tämän jälkeen suoritin komennon "slmgr /skms kms.core.windows.net" kertoakseni slmgr.vbs-työkalulle missä KMS-aktivointipalvelin sijaitseen. Tämän jälkeen pääsin suorittamaan aktivointipyyntöä komennolla "slmgr /ato", jonka jälkeen sain ilmoituksen, että aktivointi onnistui.
