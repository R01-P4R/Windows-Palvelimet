# 4. Tiedostopalvelin

## Tehtävänanto

1. Määritellään tiedostopalvelimen runko
2. Käynnistetään ohjauspalvelin ja tehdään alkumääritykset
3. Määritetään ohjauspalvelimen staattinen IP-osoite
4. Määritetään ohjauspalvelimen nimi ja poistetaan lukitusruutu-ominaisuus pois käytöstä
5. Aktivoidaan ohjauspalvelimen käyttöjärjestelmä
6. Asennetaan ohjauspalvelimelle tarvittavat roolit ja ohjelmisto

##  Tiedostopalveliment rungon määritys

Aloitin kirjautumalla Azure Labsiin ja avaamalla virtuaalikoneen. Tässä harjoituksessa luodaan uusi tiedostopalvelin virtuaaliympäristöön. Ensimmäiset askeleet ovat melkein identtiset tehtävä 3. Ohjauspalvelimen kanssa. Aloitin luomaan uutta virtuaalikonetta hyödyntäen aikaisemmassa harjoituksessa luotua virtuaalikovalevyä "WindowsServer_FileServer". Avasin Hyper-V managerin ja valitsin "New -> Virtual Machine. Annoin uudelle virtuaalikoneelle nimeksi "WindowsServer_FileServer" ja valitsin ensimmäisen generaation. Muistin määräksi asetin 3072 MB ja muistin dynaamisen allokoinnin laitoin päälle. Verkkoyhteden adapteriksi valitsin aikaisemmin luodon HyperVVSwitch ja virtuaalikovalevyksi valitsin harjoituksessa "2. Virtuaalikoneiden malli ja palvelinkoneiden alustus" luodun WindowsServer_FileServer.vhdx templaten. Lopuksi vielä muutin Hyper-V managerin kautta käytettyjen virtuaaliprosessorien ytimien määräksi 4.

## Tiedostopalvelimen käynnistys ja alkumääritykset

Aloitin yhdistämällä luotuun tiedostopalvelimeen WindowsServer_FileServer Hyper-V managerin kautta. Koneen käynnistettyä valitsin maa- ja näppäimistö asetuksiksi "Finland" ja "Finnish". Kielen muutin "English (United States)", sillä IT-maailmassa se on ns. de facto-kieli. Seuraavaksi hyväksyin lisenssiehdot, määritin ylläpitäjän salasanan kurssiohjeiden mukaisesti ja kirjauduin virtuaalikoneen työpöydälle. 


## Ohjauspalvelimen staattinen IP-osoite ja sen muuttaminen yksityiseksi

Avasin virtuaalikoneen WindowsServer_FileServer työpöydälle Server Manager-työkalun ja valikosta "Local Server" valitsin Ethernetin vierestä "IPv4 address assigned by DHCP, IPv6 enabled". Sitten valitsin Ethernet-verkkokortista "Properties" ja valitsin "Internet Protocol Version 4 (TCP/IPv4)" Tästä valikosta valitsin vielä "Properties" ja määritin ohjeen mukaisesti IPv4-tiedot (IP-addess, Subnet mask, Default Gateway ja DNS-asetukset) seuraavanlaisesti:

    IP address: 10.208.0.12
    Subnet mask: 255.255.255.0
    Default Gateway: 10.208.0.1
    Preferred DNS sever: 10.208.0.10  <- Ohjauspalvelimen IP-osoite

Nämä asetukset määritettyä sain Windowsilta ilmoituksen uuden verkon luomisesta ja verkkotilatyypin määrityksestä. Tässä tehtävässä määritin sen manuaalisesti yksityiseksi. Aloitin avaamalla cmd:n ylläpitäjänä ja kirjoitin komennon: "gpedit.msc", mikä avaa kyseisen ohjelman, jonka avulla voi määrittää ryhmäkäytäntöjä. Muokkasin kohdasta: "Computer Configuratio" -> "Windows Settings" -> "Security Settings" -> "Network List Manager Policies" -> "Network" ja hiiren oikealla "Properties". Valitsin valikosta "Private" ja hyväksyin muutokset.


Lopuksi testasin verkkoyhteyden toimivuutta cmd:en kautta komennolla "nslookup haaga-helia.fi". Sain toivetun tuloksen.

## Ohjauspalvelimen nimi ja lukitusruudun poistaminen käytöstä

Aloitin valitsemalla virtuaalikoneen WindowsServer_DC Server Managerin ja valikosta Local Server valitsin luodon tietokoneen nimen. Kohdasta "Computer Name" valitsin "Change..." ja määritin ohjeiden mukaisesti uuden nimen "Controller-Partanen" ohjauspalvelimelle. Nimen muutoksen jälkeen oli virtuaalikoneen käynnistettävä uudestaan.

## Ohjauspalvelimen käyttöjärjestelmän aktivointi

Aloitin avaamalla virtuaalikoneen WindowsServer_DC:n komentorivin (cmd) ylläpitäjän oikeuksin ja suorittamalla komennon "slmgr.vbs", joka antaa infoa Windows käyttöjärjestelmän aktivoitamiseen käytettävän työkalun slmgr.vbs käytöstä. Läpikäytyäni tiedot suoritin komennon "slmgr.vbs /dlv" saadakseni tietää nykyaktivoinnin tilan. Tämän jälkeen suoritin komennon "slmgr /skms kms.core.windows.net" kertoakseni slmgr.vbs-työkalulle missä KMS-aktivointipalvelin sijaitseen. Tämän jälkeen pääsin suorittamaan aktivointipyyntöä komennolla "slmgr /ato", jonka jälkeen sain ilmoituksen, että aktivointi onnistui.

## Ohjauspalvelimen rooli ja palvelinohjelmisto

Aloitin käynnistämällä virtuaalikoneen WindowsServer_DC:n ja Server Managerin kautta valitsin oikeasta yläkulmasta "Manage" ja "Add Roles and Features". Avautuvan infon luettuani valitsen "Role-based or feature-based installation" ja valitsin listalta ohjauspalvelimen Controller-Partanen. Valitsin tarjottavista rooleista "Active Directory Domain Services" valitsin seuraavasta ikkunasta "Include management tools" ja lopuksi siirryin kohtaan "Add Features". Features kohdasta siirryin oletusasetuksilla eteenpäin. Sain tietoa asennuksessa valituista rooleista ja yhteenvedon valituista ominaisuuksista. Lopuksi varmistin asennuksen ja virtuaalikone käynnistyi uudestaan.

<br>
<br>

Asennuksen onnistuttua avasin Server Managerin ja uuden valikon "AD DS" alta valitsin "Controller-Partanen". Valitsin kohdasta "More..." -> "Promote this server to a domain controller". Aukeavasta valikosta pääsin luomaan aktiivihakemistolle uutta ylintä tasoa eli "forest", valitisn "Add a new forest" ja asetin sen nimeksi "partanen.lan". Valitsin seuraavaksi Windows Server 2016-tason, ja valitsin myös "Domain Name System (DNS) Server", eli ohjauspalvelimella Controller-Partanen on nimipalvelimen (DNS) rooli. DSRM-salasanan asetin ohjeen mukaan. DSRM on toimnto jolla kone voidaan käynnistää, vaikka aktiivihakemistoon tulisi häiriö. NetBios-toimialeen mimeksi valitsin päätoimialueen nimen ensimmäisen osan isolla kirjoitettuna, eli "PARTANEN". Seuraavan ikkunan polkuvaihtoehdot pidin oletuksena. Asennuksen lopuksi tietokone käynnistyi uudestaan.
