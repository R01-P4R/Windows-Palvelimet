# 6. Ryhmäkäytännöt

## Tehtävänanto

1. Määritellään tiedostopalvelimen runko
2. Käynnistetään tiedostopalvelin ja tehdään alkumääritykset
3. Määritetään tiedostopalvelimen staattinen IP-osoite
4. Määritetään tiedostopalvelimen nimi ja poistetaan lukitusruutu-ominaisuus pois käytöstä
5. Aktivoidaan tiedostopalvelimen käyttöjärjestelmä
6. Asennetaan tiedostopalvelimelle tarvittavat roolit ja ohjelmisto

##  Valmistelevat toimenpiteet

Aloitin varmistamalla ensin, että molemmat virtuaalikoneet WindowsServer_DC ja WindowsServer_FileServer olivat päällä taustalla Hyper V-managerissa. Tämän jälkeen yhdistin virtuaalikoneseen WindowsServer_DC ja hyödyntäen Server Manageria siirryin muuttamaan tiedostopalvelimen (WindowsServer_FileServer) asetuksia kohdasta "All Servers" -> valitsemalla tiedostopalvelimen ja "Computer Management".

## Tiedostopalvelimen käynnistys ja alkumääritykset

Aloitin yhdistämällä luotuun tiedostopalvelimeen WindowsServer_FileServer Hyper-V managerin kautta. Koneen käynnistettyä valitsin maa- ja näppäimistö asetuksiksi "Finland" ja "Finnish". Kielen muutin "English (United States)", sillä IT-maailmassa se on ns. de facto-kieli. Seuraavaksi hyväksyin lisenssiehdot, määritin ylläpitäjän salasanan kurssiohjeiden mukaisesti ja kirjauduin virtuaalikoneen työpöydälle. 


## Tiedostopalvelimen staattinen IP-osoite ja sen muuttaminen yksityiseksi


<br>

    


<br>

Nämä asetukset määritettyä sain Windowsilta ilmoituksen uuden verkon luomisesta ja verkkotilatyypin määrityksestä. Tässä tehtävässä määritin sen manuaalisesti yksityiseksi. Aloitin avaamalla cmd:n ylläpitäjänä ja kirjoitin komennon: "gpedit.msc", mikä avaa kyseisen komennon nimisen ohjelman, jonka avulla voi määrittää ryhmäkäytäntöjä. Muokkasin kohdasta: "Computer Configuration" -> "Windows Settings" -> "Security Settings" -> "Network List Manager Policies" -> "Network" ja hiiren oikealla "Properties". Valitsin valikosta "Private" ja hyväksyin muutokset. Tarkistin, että virtuaalikone WindowsServer_DC oli päällä taustalla ja suortin cmd:ssä komennon: "nslookup Haaga-helia.fi". Sain ohjeen mukaisen vastauksen, joten oletettavasti yhteydet toimivat ja DNS-palvelimen IP-osoite on oikein.

<br>

Lopuksi tarkastin virtuaalikoneen WindowsServer_FileServer Server Managerista, että palomuuri- ja osoiteasetukset olivat muuttuneet halutuksi.


## Tiedostopalvelimen nimi ja lukitusruudun poistaminen käytöstä

Aloitin valitsemalla virtuaalikoneen WindowsServer_FileServer Server Managerin ja valikosta Local Server valitsin luodon tietokoneen nimen. Kohdasta "Computer Name" valitsin "Change..." ja määritin ohjeiden mukaisesti uuden nimen "FileServer-Partanen" tiedostopalvelimelle. Nimen muutoksen jälkeen oli virtuaalikoneen käynnistettävä uudestaan.

## Tiedostopalvelimen käyttöjärjestelmän aktivointi

Aloitin avaamalla virtuaalikoneen WindowsServer_FileServer:n komentorivin (cmd) ylläpitäjän oikeuksin ja suoritin komennon: "slmgr /skms kms.core.windows.net" kertoakseni slmgr.vbs-työkalulle missä KMS-aktivointipalvelin sijaitseen. Tämän jälkeen pääsin suorittamaan aktivointipyyntöä komennolla "slmgr /ato", jonka jälkeen sain ilmoituksen, että aktivointi onnistui.



## Tiedostopalvelimen liittäminen toimialueeseen

Aloitin tarkistamalla, että molemmat koneet olivat päällä Azure-virtuaalikoneen Hyper-V Managerissa, minkä jälkeen yhdistin virtuaalikoneeseen WindowsServer_FileServer ja Server Managerissa valitsin: "Local Server" ja "WORKGROUP". Kohdasta "Change..." pääsin lisäämään virtuaalikoneen toimialueeseen "partanen.lan". Tämä piti hyväksyä toimialueen, eli ohjauspalvelimen ylläpitäjän tunnuksin. Liittymisen jälkeen oli WindowsServer_FileServer käynnistettävä uudestaan. Lopuksi vielä kokeilin kirjautua toimialueen ylläpitäjän tunnuksin tiedostopalvelin koneelle ja se onnistui.

##  Tiedostopalvelun asentaminen ohjauspalvelimelta




Aloitin avaamalla WindowsServer_DC virtuaalikoneen Server Managerin ja kohdasta "Manage" valitsin "Add Servers".

<br>


<img width="931" height="589" alt="image" src="https://github.com/user-attachments/assets/f2e0e6cb-15e1-4b5d-8a28-f6112a176c74" />

<br>


Active Directory-välilehdeltä hain aikaisemmin luodun tiedostopalvelimen haulla: "FILESEVER" ja lisäsin sen tunnettujen laitteiden listaan. Siirryin takaisin Server Managerin All Servers-näkymään ja valitsin tiedostopalvelimeni. Kohdasta "Manage" -> "Add Roles and Features" pääsin lisäämään tiedostopalvelimen roolit ja ominaisuudet valitsemalla ensin "Role-based or feature-based installation" seuraavalla sivulla luodun tiedostopalvelimeni ja sitten seuraavat roolit: "File Server" ja  "File Server Resource Manager" ("Include management tools" valinta myös avautuvassa ikkunassa) näiden roolien kanssa. Select Features-vaihtoehdot jätin oletukseksi. Viimeisellä sivulla valitsin "Restart the destination server automatically if required" ja painoin "install". Asennuksen lopuksi käynnistin tiedostopalvelimen (WindowsServer_FileServer) uudestaan hyödyntäen ohjauspalvelimen (WindowsServer_DC) Server Manageria.


## Verkkojakojen luonti

Aloitin kirjautumalla tiedostopalvelimeen toimialueen ylläpitäjän tunnuksin ja avasin "Server Manager" -> "File and Storage Services".

<br>

<img width="696" height="439" alt="image" src="https://github.com/user-attachments/assets/ee473277-26ba-4a67-a5b8-ec647d3c4adf" />

<br>

Loin uuden jaon (share) asennusohjelman avulla. Valitsin ensin "SMB Share – Quick" ja seuraavassa kohdassa määritin uuden polun "C:\Files", mikä luo mnyös tarvittavan kansion, jos sitä ei vielä ole. Verkkolevyjaolle annoin nimeksi "Data" ja valitsin seuraavaksi "Allow caching of share". Seuraavaksi pääsin tarkastelemaan luotavan verkkojaon oikeuksia ja yhteenvetoa valituista asetuksista hyväksyin ne muuttamatta ja painoin "install" ja "close". 

<br>

Siirryin ohjauspalvelimen Server Manageriin seuraavaksi ja kohdasta "All Servers" valitsin tiedostopalvelimen ja "Computer Management". Sain ilmoituksen, että tiedostopalvelimen palomuuriasetukset estävät pääsyn etäynä. Siirryin takaisin tiedostopalvelimelle ylläpitäjän tunnuksin ja muutin asetuksia kohdasta: "Windows Defender Firewall with Advanced Security" -> "Inbound Rules" seuraavat säännöt hyväksytyksi: Remote Event Log Management (NP-In), Remote Event Log Management (RPC) ja Remote Event Log Management (RPC-EPMAP). Tämä avaa tiedostopalvelimen portit ohjauspalvelimen työkaluille.

Asetuksia muuttamalla pääsin ohjauspalvelimelta yhdistämään etänä tiedostopalvelimelle ja tarkastamaan sen hallintanäkymää, sekä tiedostoja. Seuraavaksi loin ohjauspalvelimelta käsin uuden jaon etänä tiedostopalvelimelle avaamalla ensin Server Managerin ja tiedostopalvelimen nimen kohdalta: "Computer Management". Polussa "Shared Folders" -> "Shares" -> "New Share..." pääsin luomaan uutta jakoa asennusohjelman avulla. Loin uuden kansion "C:\MManagement" aikaisemmin luodon kansion "Files" alapuolelle ja jaon nimeksi annoin "Person" asetuksella: "No files or programs from the shared folder are available offline". Lopuksi määritin vielä verkkojaolle vähimpien oikeuksien periaatteella "Administrators have full access; other users have no access". Uuden jaon luominen onnistui.
