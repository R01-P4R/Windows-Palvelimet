## DNS forwarders

<br>

Tehtävässä oli tarkoitus tutustua DNS-palveluun ja DNS forwarders -nimipalvelimiin. Aloitin kirjautumalla toimialueeni ohjauspalvelimelle (WindowsServer_DC) ja Server Managerista valitsin: "Tools" -> "DNS". Seuraavaksi valitsin ohjauspalvelimeni nimen alta: "Forwarders". Forwardersien alta löytyi Azure Labsin käytössä oleva DNS-forwarder "168.63.129.16". DNS-forwadrien tarkoitus on toimia palvelimena, joka välittää DNS-pyyntöjä eteenpäin ulkopuolisille DNS-palvelimille, mikäli se ei pysty selvittään niitä paikallisesti. 

<br>

## DNS-välimuistit

<br>

Tehtävässä oli tarkoitus tutustua DNS-palvelun välimuistin toimintaan verkon osana ja mitä tietoja eri DNS-rooleissa olevat tietokoneet tallentavat. Toimialueellaani ohjauspalvelin (WindowsServer_DC) toimii DNS-palvelimena ja tiedostopalvelin (WindowsServer_FileServer) toimii DNS-asiakkaana. Tärkeää oli, että molemmat virtuaalikoneet olivat päällä harjoituksen aikana.

Testasin aluksi asiakkaana toimivan tiedostopalvelimen välimuistia DNS-asiakkaana avaamalla ensin tiedostopalvelimen CMD:n ja käyttämällä komentoa:                               
                           
                                      nslookup wikipedia.org

Komento suorittaa DNS-kyselyn annetusta verkkosivusta. Vastauksena (Non-authorative answer) sain kyseisen sivuston käyttämät IP-osoitteet. Nslookup-komento on hyödyllinen työkalu DNS-palvelun toimivuuden testaamiseen. Vaustauksessa näkyi, että ohjauspalvelimeni toimii nimipalvelimena (asensin aikaisemmassa tehtävässä nimipalvelimen roolin ohjauspalvelimelle). Non-authorative answer tarkoittaa, että DNS-kysely suoritettiin DNS-palvelimelle, joka ei ole kyseisen domainin päälähde, minkä seurauksena DNS-palvelin vastaa toisen DMS-palvelimelta saaduilla tiedoilla.

<br>

 > Ryhmäkäytäntöjen mallinnus määritysten työkalu.

<br>

Ensimmäisellä valintaruudulla valittiin toimialueen ohjauspalvelin, minkä avulla mallinnus toteutetaan. Valitsin toimialueeni ohjauspalvelimen. Seuraavalla sivulla määritettiin ryhmäkäytäntöjen mallinnuksen kohde (käyttäjä tai tietokone), valitsemalla se aktiivihakemiston säiliö, missä kohteena oleva resurssi sijaitsee. Harjoituksen tarkoituksena oli suorittaa mallinnus aikaisemmassa tehtävässä luoduille käyttäjille: "Prod Uction" ja "prodnon Uction" pöytätietokoneella, joka sijaitsee kuvitteellisesti Desktops-organisaatioyksikössä. Valinnassa käytettiin suoria LDAP-polkuviittauksia (Lightweight Directory Access Protocol) seuraavanlaisesti:

    User information
    Container:
    OU=Production,OU=Accounts,DC=partanen,DC=lan

    Computer information
    Container:
    OU=Desktops,OU=Resources,DC=partanen,DC=lan

## Kysymyksiä:

### Mikä on RSOP? Mihin sitä käytetään ja mitä hyötyä siitä on?

RSOP (Resultant Set of Policy) on aktiivihakemiston työkalu, minkä avulla ylläpitäjät pystyvät keräämään tietoa käytössä olevista ryhmäkäytännöistä, sekä suorittaa vianmääritystä ympäristön tietokoneiden ja käyttäjien ryhmäkäytäntöjen toimivuudesta.

### Minkä tyyppistä tietoa aktiivihakemiston hakemistotietokantaan kohdistettavalla LDAP-kyselyllä saadaan? Anna esimerkkejä.

LDAP-kyselyllä (Lightweight Directory Access Protocol) voidaan hakea tietoa mm. toimialueen käyttäjistä, ryhmistä ja jäsenyyksistä, sekä tietokoneista ja laitteista.
